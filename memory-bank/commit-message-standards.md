# 提交信息规范 (Commit Message Standards)

## 📋 总体格式

```
🔧 [工作类型]: [成果概述]

🚀 重大功能完成/重大突破成果:
- 具体功能点1
- 具体功能点2
- 具体功能点3

📦 技术实现突破/技术实现:
- 技术实现细节1
- 技术实现细节2
- 技术实现细节3

🎯 解析能力/验证能力达成:
- ✅ 验证项1 详细描述
- ✅ 验证项2 详细描述
- ✅ 验证项3 详细描述

📊 状态: [完成状态描述，生产就绪程度，后续计划]
```

## 🎯 标题格式规范

### Emoji 类型映射
- 🎨 **创意阶段**: UI/UX优化、设计改进
- 🎉 **重大完成**: 核心功能验证、里程碑达成
- 🔧 **技术修复/优化**: 问题修复、配置优化、技术改进
- 🚀 **新功能**: 重大功能添加
- 📋 **文档/任务**: 文档更新、任务状态变更
- 🤖 **环境配置**: 开发环境、CI/CD配置
- ✅ **阶段完成**: 实施阶段、测试阶段完成
- 🧪 **测试相关**: 测试框架、测试用例添加

### 标题结构
`🔧 [工作类型/状态]: [具体成果概述]`

**示例**:
- `🎨 创意阶段UI/UX优化完成: 分层优化方案实施`
- `🎉 完成API Navigator核心功能验证: 独立面板+精确跳转全面就绪`
- `🔧 修复核心质量问题: JavaASTParser解析引擎与测试框架重构`
- `🔧 优化插件打包配置: .vscodeignore精简57%安装包体积`

## 📝 详细描述规范

### 1. 🚀 重大功能完成/重大突破成果
- 列出核心完成的功能
- 突出关键技术突破
- 强调用户价值和体验改进

### 2. 📦 技术实现突破/技术实现
- 详细技术实现细节
- 新增的类、方法、配置
- 架构改进和优化点

### 3. 🎯 解析能力/验证能力达成
- 使用 ✅ 格式列出验证项
- 包含具体的性能指标
- 覆盖功能完整性验证

### 4. 📊 状态
- 当前完成度（百分比）
- 生产就绪程度
- 后续计划或能力描述

## ⚡ 使用指南

1. **标题精炼**: 控制在50字符以内，突出核心成果
2. **详细描述**: 每个section 3-5个要点，具体量化
3. **验证完整**: 🎯部分必须包含功能验证，使用✅标记
4. **状态明确**: 📊部分说明完成度和后续计划
5. **技术价值**: 突出技术实现和用户价值

## 🔍 质量标准

- **可读性**: 技术和非技术人员都能理解
- **完整性**: 涵盖功能、技术、验证、状态四个维度
- **量化**: 包含具体的数字和性能指标
- **追溯性**: 能够通过提交信息快速理解变更内容
- **一致性**: 遵循统一的格式和emoji使用规范

### 📋 Release Note 编写指南

#### 版本主题命名
- **重大功能**: "多功能增强"、"架构升级"、"生态扩展"
- **重要修复**: "核心修复"、"解析增强"、"稳定性提升"
- **功能优化**: "体验优化"、"性能提升"、"界面改进"
- **技术完善**: "工程化改进"、"质量提升"、"规范化"

#### 核心功能分组
1. **按技术领域**: 解析引擎、UI界面、缓存系统、CI/CD
2. **按用户价值**: 搜索体验、代码导航、项目分析、开发效率
3. **按开发阶段**: 新增功能、功能增强、问题修复、性能优化
4. **按影响范围**: 核心功能、辅助功能、开发体验、用户体验

#### 量化数据要求
- **性能提升**: 具体百分比或倍数
- **体积优化**: 具体 MB 数或百分比
- **功能覆盖**: 支持的注解数量、HTTP 方法数
- **测试覆盖**: 测试用例数量、通过率
- **用户体验**: 操作步骤减少、响应时间提升

#### 文案风格标准
- **专业性**: 使用准确的技术术语
- **用户导向**: 突出用户价值和体验改进
- **量化表达**: 用数据说话，避免主观描述
- **简洁明了**: 每个要点控制在一行内
- **一致性**: 同类功能使用统一的描述格式

#### 版本对比链接规范
- **格式**: `https://github.com/xkcoding/API-Navigator/compare/v{上一版本}...v{当前版本}`
- **用途**: 方便用户查看版本间的具体代码变更
- **替换规则**: 每次发版时替换版本号
- **示例**: 
  - v1.0.5 → v1.0.6: `compare/v1.0.5...v1.0.6`
  - v1.0.6 → v1.0.7: `compare/v1.0.6...v1.0.7`
- **位置**: 放在完整更新日志和问题反馈之间
